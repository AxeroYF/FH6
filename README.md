# FH6Auto v4.1

FH6Auto 是一个基于 Python、图像识别和输入自动化的 FH6 本地辅助工具。

当前版本重点支持：

- 循环跑图
- 批量买车
- 超级抽奖
- 大循环串联
- AI 辅助选车
- 诊断记录
- 启动自适应校准

> 本项目仅用于自动化、图像识别和本地脚本技术交流学习。请勿用于商业用途或破坏游戏平衡。使用风险请自行承担。

## 项目来源

本项目基于 [YOUSTHEONE/FH6Auto](https://github.com/YOUSTHEONE/FH6Auto) 持续维护，并结合当前游戏版本和实际用户环境做了适配、重构与稳定性优化。

## v4.1 更新内容

### 1. 结构重构

- 将原本集中在 `main.py` 的三条业务流程拆分为：
  - `flow_race.py`
  - `flow_buy.py`
  - `flow_cj.py`
- 新增 `flow_common.py` 复用公共流程逻辑。
- 主程序结构更清晰，后续排障和迭代更容易。

### 2. 图片资源目录整理

- 所有模板图片统一归档到 `images/1080p/`
- 程序内所有相关路径同步更新
- 额外更新了离线工具脚本，兼容新的图片目录结构

### 3. 启动自适应校准

- 软件启动后会自动检测游戏窗口信息
- 根据窗口尺寸、亮度、清晰度和锚点模板结果，动态调整：
  - 模板缩放优先级
  - 灰度识别阈值偏移
- 主界面新增“自适应校准”状态显示，用户可以直接看到当前校准状态

### 4. 识图与流程优化

- 优化模板缩放搜索策略，优先尝试窗口推导比例和校准结果
- 优化部分灰度识别、透明通道识别和多重匹配日志
- 清理 `flow_race` 中作者点赞弹窗的重复扫描
- 对 `flow_race` 找不到 `liketag` 车辆时的等待进行了保守提速，不改变原流程结构

### 5. 诊断记录模式

- 新增“诊断记录”开关
- 开启后，程序正常执行原流程，同时记录真实运行中的识图命中与未命中
- 每轮结束后会在 `debug/diagnostics/<timestamp>/` 下输出：
  - `events.jsonl`
  - `summary.json`
  - `report.txt`
- `report.txt` 为玩家可读格式，适合反馈问题和复盘

### 6. 批量买车 CR 限制

- 在批量买车区域新增 `CR` 输入框
- 总买车次数限制规则：

```text
总买车上限 = floor(CR / 86000)
```

- 该限制作用于整个任务运行周期，而不是单轮买车
- 当买车额度不足或耗尽时：
  - 会自动限制本轮买车数量
  - 或跳过买车直接进入超抽
  - 超抽完成后停止整个循环，避免已买新车浪费

## 运行环境建议

- Windows
- 游戏语言：简体中文
- 输入法：英文键盘
- 建议分辨率环境：1080P
- 建议关闭 HDR
- 运行时保持游戏在前台

## 功能说明

### 循环跑图

- 自动进入 EventLab
- 自动输入蓝图代码
- 自动寻找赛事入口并重复执行

### 批量买车

- 自动进入车辆收藏和目标车辆页面
- 自动重复购买
- 支持基于 CR 数量的总额度限制

### 超级抽奖

- 自动进入车辆菜单、选车、上车、点技能
- 支持 AI 辅助选车
- 支持智能页码

### 模块串联

支持以下大循环链路：

```text
跑图 -> 买车 -> 超抽 -> 下一轮
```

每个模块完成后是否继续，均可在主界面中配置。

## 常用开关

- `AI辅助`：启用 AI 参与选车
- `智能页码`：启用页码记忆跳转
- `纯AI`：只使用 AI 识别，不回退模板
- `自动截图`：保存 AI 识别样本
- `诊断记录`：保存完整诊断报告

## 快捷键

- `F8`：停止任务
- `F3`：测试找图流程

## 从源码运行

标准环境：

```powershell
python -m venv .venv
.\.venv\Scripts\activate
python -m pip install -r requirements.txt
python main.py
```

AI 环境：

```powershell
python -m pip install -r requirements-ai.txt
```

模型文件默认路径：

```text
models/fh6_car_select_yolo.pt
```

## 补充说明

如果识图兼容性不理想，优先检查：

- 分辨率 / 缩放
- HDR
- 颜色管理
- 游戏语言
- 模板图片是否与当前界面一致

需要复盘问题时，建议先开启“诊断记录”再跑一轮。
