# FH6Auto v4.2.1 更新公告

发布日期：2026-07-19

## v4.2.1 修正内容

- 修正流程设置中连接关系错误和选项冗余的问题。流程设置现在只保留四个相邻连接：跑图 → 买车、买车 → 超级抽奖、超级抽奖 → 删除车辆、删除车辆 → 跑图。
- 移除旧版数字跳转输入、“超级抽奖 → 跑图”和“跑图 → 删除车辆”等错误路线，界面显示与调度器执行逻辑保持一致。
- 四个连接全部开启时，完整大循环固定按照“跑图 → 买车 → 超级抽奖 → 删除车辆 → 跑图”执行；只有删除车辆完成并返回跑图时才计为完成一轮。
- 关闭任意连接后，当前模块完成即停止，不再跳转到非相邻模块。
- 自动迁移 v4.2 的旧流程配置，清理已废弃的跳转字段，避免旧配置继续触发错误路线。
- 正式拆分 Steam 与 Xbox 两套程序入口和发布包。Steam 版继续使用游戏内分享码输入，Xbox 版使用 Xbox 系统输入窗口兼容层；公共业务流程修正会同步应用于两个版本。

## v4.2.1 发布包

- Steam：`FH6Auto-4.2.1-PureAI`
- Xbox：`FH6Auto-4.2.1-Xbox-PureAI`

# FH6Auto v4.2 更新公告

发布日期：2026-07-18

## v4.2 核心更新

- 跑图流程适配新版 EventLab 挑战关卡，完善挑战结果判断、失败轮次计数、末轮退出及评分点踩处理。
- 保留斯巴鲁 22B 方案，并新增 Mazda #123 Mad Mike 808 Wagon 可选方案；买车、超级抽奖、车辆价格和默认技能树路径会随车辆选择自动联动。
- Mazda 808 正式启用独立 YOLO 选车模型，提升超级抽奖流程中的目标车辆识别稳定性。
- 新增“删除车辆”模块，可按设定数量筛选并删除 Mazda 808，并已接入完整大循环。
- 完整大循环现支持：跑图 → 买车 → 超级抽奖 → 删除车辆 → 跑图。
- 新增后台鼠标、后台键盘与窗口画面捕获支持，减少脚本运行时对物理鼠标的占用，并支持运行时自动缩小游戏窗口。
- 重做并压缩主界面布局，完善车辆切换、流程编排、守护设置和各模块状态展示。
- 精简前台运行日志，仅保留运行须知、模式状态、AI 加载、流程起止、循环进度和关键错误；详细识别信息保留在诊断日志中。
- 新增项目逻辑说明与流程图，便于理解各模块及完整循环的运行关系。

## 车辆参数

- 斯巴鲁 22B：330,000 CR。
- Mazda 808：95,000 CR，需要对应通行证权限。

## 使用提醒

- 请将游戏语言设为简体中文，并保持游戏内按键设置与项目要求一致。
- 后台运行能力依赖游戏窗口和系统环境；如游戏在失去焦点后主动暂停，请先关闭相关暂停设置。
- 建议首次使用新车辆方案或完整大循环时开启诊断模式，以便出现异常时快速定位。

## v4.1 更新说明

本次 `v4.1` 是一次幅度较大的整理与升级，重点不是单点修补，而是把结构、识图、诊断和运行兼容性一起往前推进了一步。

## v4.1 主要更新

### 1. 主流程重构

- 将原本集中在 `main.py` 的三条业务流程拆分为：
  - `flow_race.py`
  - `flow_buy.py`
  - `flow_cj.py`
- 新增 `flow_common.py` 复用公共逻辑。

这次拆分后，后续继续排查跑图、买车、超抽问题会更直接，维护成本也更低。

### 2. 图片资源统一整理

- 所有模板图片迁移到 `images/1080p/`
- 程序内相关引用路径同步更新
- 离线工具脚本也已兼容新的资源目录结构

这一步主要是为了给后续继续扩展资源方案和分辨率适配打基础。

### 3. 启动自适应校准

- 软件启动时自动检测游戏窗口信息
- 根据窗口尺寸、亮度、清晰度和锚点模板结果，动态调整模板缩放优先级和灰度阈值
- 主界面新增“自适应校准”状态显示

这项更新的目标是降低不同客户端环境下的模板识别脆弱性。

### 4. 识图与流程优化

- 优化模板缩放搜索顺序
- 清理 `flow_race` 中作者评价弹窗的重复扫描
- 保守压缩部分无效等待时间
- 修正和补充部分识图日志输出

整体方向是减少拖沓感，同时保持现有流程逻辑不变。

### 5. 诊断记录模式

- 新增“诊断记录”开关
- 程序按原流程正常运行，同时旁路记录识图命中与未命中
- 每轮结束后输出：
  - `events.jsonl`
  - `summary.json`
  - `report.txt`

其中 `report.txt` 是玩家可读格式，适合反馈问题和复盘。

### 6. 批量买车 CR 限制

- 批量买车区域新增 `CR` 输入框
- 总买车上限规则：

```text
总买车上限 = floor(CR / 86000)
```

- 当额度耗尽时：
  - 跳过后续买车
  - 自动转入超抽
  - 超抽完成后停止整个循环

这样可以尽量避免已买新车被浪费。

`v4.1` 的核心价值，不是单独新增某一个功能，而是：
